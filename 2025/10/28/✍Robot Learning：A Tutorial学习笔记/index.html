Chapter1 Introduction

本教程的目标是双重的：一方面为机器人学习领域中的常见技术提供科学原理和实际应用的有用参考，另一方面为研究人员和从业者展示如何具体应用这些技术。为此，我们力求简明而严谨地概述所介绍技术背后的核心概念，并结合 lerobot 中的代码示例，提供如何实际使用这些技术的案例。本教程的结构如下：

  • 第2节 回顾经典的机器人学基础，介绍基于动力学方法的局限性。
  • 第3节 进一步阐述基于动力学方法的局限性，并介绍强化学习（RL）作为解决机器人问题的实际方法，讨论其优点和潜在局限。
  • 第4节 详细描述旨在解决单任务学习的机器人学习技术，利用行为克隆（BC）技术自主地复现特定的专家示范。
  • 第5节 介绍在开发通用机器人学习模型方面的最新进展，这些模型通过从大规模的多任务和多机器人数据中学习（即机器人基础模型）。

本教程的目标是为读者提供直观的解释，帮助理解为什么来自机器学习（ML）领域的各种看似分散的理念会逐渐汇聚，并推动当前机器人学的迅速发展，从而实现前所未有的进步。我们在展示机器人学习领域中最常见和最新的研究方法时，将结合 lerobot 中的实际代码实现，并首先介绍 lerobot 中引入的数据集格式。

图1 | lerobot 是由 Hugging Face 开发的开源端到端机器人库。该库在整个机器人技术栈上进行了垂直集成，支持对真实机器人设备的低级控制、先进的数据和推理优化，以及使用纯 PyTorch 实现的最先进的机器人学习方法。

1 LeRobotDataset

LeRobotDataset 是 lerobot 中的重要功能，专为机器人学习研究设计，旨在应对机器人数据日益重要的现状。它定义了一个标准化的数据集格式，便于统一访问多种传感器数据，包括传感器运动数据、多路摄像头视频流和遥控状态等。它还能够存储数据采集的相关信息，如任务描述、机器人种类、图像和机器人状态流的帧率等。

该数据集格式提供了一个统一的接口，用于处理多模态的时间序列数据，并与 PyTorch 和 Hugging Face 生态系统无缝集成。LeRobotDataset 支持用户扩展和高度自定义，能够处理来自多种机器人平台（如 SO-100 臂、ALOHA-2 设置、真实世界的人形臂与手、模拟数据集以及自动驾驶汽车）的开放数据。

1.1 LeRobotDataset 数据集类设计

LeRobotDataset 的核心设计选择之一是将底层数据存储与面向用户的 API 分离。这种设计方法允许在提供直观且易于使用的数据格式的同时，保持高效的数据存储。

数据集总是被组织为以下三个主要部分：

  • 表格数据 (Tabular Data)：低维度、高频率的数据，如关节状态和动作，存储在高效的内存映射文件中，通常会被传输到 Hugging Face 的成熟数据集库中，这样可以提供快速访问并减少内存消耗。
  • 视觉数据 (Visual Data)：为了处理大量的摄像头数据，视频帧会被串联并编码成 MP4 文件。同一集（episode）的帧始终被聚集在一个视频文件中，并且按摄像头分组多个视频。为了减轻文件系统的压力，属于同一摄像头视角的多个视频文件会被分成多个子目录，超出一定阈值后会自动划分。
  • 元数据 (Metadata)：一组 JSON 文件，用于描述数据集的结构，作为表格和视觉数据的关系部分。元数据包括不同的特征架构、帧率、归一化统计信息和集的边界。

🤗可扩展性和大规模支持

为了支持可能包含数百万轨迹的大型数据集（导致数亿或数十亿个摄像头帧），我们将不同集的数据合并到相同的高层结构中。具体来说，这意味着任何给定的表格数据集合和视频文件通常不会仅包含一个集的数据，而是包含多个集的数据拼接。这种方法能够减轻文件系统的压力，无论是本地存储还是像 Hugging Face 这样的远程存储提供商，但也需要依赖更关系型的元数据部分来重建信息，如文件中的某一集开始和结束的位置。

😻LeRobotDataset 示例结构

  • meta/info.json：这是元数据的核心文件，包含完整的数据集架构，定义所有特征（如观察状态、动作）、它们的形状和数据类型。它还存储重要信息，如数据集的帧率、数据采集时的 lerobot 版本，以及用于定位数据和视频文件的路径模板。
  • meta/stats.json：该文件存储了数据集中每个特征的聚合统计信息（均值、标准差、最小值、最大值），用于大多数策略模型的数据归一化，并且可以通过 dataset.meta.stats 进行访问。
  • meta/tasks.jsonl：这个文件包含自然语言任务描述与整数任务索引之间的映射，方便任务条件的策略训练。
  • meta/episodes/：该目录包含每个单独集的元数据，如集的长度、相应的任务以及指向数据在数据集文件中存储位置的指针。为了可扩展性，这些信息被存储在多个文件中，而不是一个庞大的 JSON 文件。
  • data/：包含逐帧的表格数据，使用 parquet 文件以实现快速的内存映射访问。为了提高性能并处理大规模数据集，来自多个集的数据会合并到更大的文件中，并将这些文件组织成分块子目录，以便更好地管理目录大小。单个文件通常包含多个集的数据。
  • videos/：包含所有视觉观测流的 MP4 视频文件。与 data/ 目录类似，来自多个集的视频会被合并为单个 MP4 文件。这种策略显著减少了数据集中的文件数量，这对于现代文件系统更为高效。

LeRobotDataset 的设计旨在高效存储和管理大量的机器人学习数据，同时保持良好的扩展性。通过将表格数据、视觉数据和元数据分开组织，并利用内存映射和视频文件合并的策略，确保了数据存储和访问的高效性。

1.2 代码示例：批处理（流式）数据集

本节介绍如何使用 LeRobotDataset 类访问 Hugging Face 上托管的数据集。任何包含三个主要组成部分（表格数据、视觉数据和关系元数据）的数据集，都可以通过单一指令访问。

在实际应用中，大多数强化学习（RL）和行为克隆（BC）算法通常会处理一系列的观察和动作。为了简化起见，我们将关节空间和摄像头帧统称为 “帧”。例如，RL 算法可能会使用过去帧的历史（例如，ot−Ho:t）来缓解部分可观测性问题，而 BC 算法通常训练以回归多个动作的块（例如，at+t+Ha），而非单一控制动作。

🙌LeRobotDataset 的窗口化操作

为了适应机器人学习训练的这些特点，LeRobotDataset 提供了原生的窗口化操作，允许用户定义围绕给定帧的时间窗口（前后几秒），通过 delta_timestemps 功能。对于不可用的帧，会进行适当的填充，同时返回一个填充掩码，用于筛选出填充的帧。值得注意的是，这些操作都发生在 LeRobotDataset 内部，对上层训练 ML 模型常用的封装器（如 torch.utils.data.DataLoader）完全透明。

👍与 PyTorch DataLoader 的结合

通过将 LeRobotDataset 与 PyTorch 的 DataLoader 结合使用，用户可以自动将数据集中的单个样本字典合并为一个包含批量张量的字典，方便下游训练或推理。

🤗流式数据集支持

LeRobotDataset 还原生支持流式数据集模式。用户只需在实现中进行一行简单的修改，就可以从 Hugging Face Hub 流式获取大型数据集。流式数据集支持高性能的批处理（大约每秒 80-100 次迭代，具体取决于连接情况）和高水平的帧随机化，这是实际 BC 算法的重要特性，否则可能会很慢或基于高度非独立同分布（非 i.i.d.）的数据进行训练。

该特性旨在提升可访问性，使得用户可以在不需要大量内存和存储的情况下处理大型数据集。

  • LeRobotDataset 提供了简便的数据集访问方法，可以从 Hugging Face Hub 获取包含表格数据、视觉数据和元数据的数据集。
  • 支持原生窗口化操作，可以根据给定帧的前后时间窗口进行数据处理，并且支持填充和掩码操作。
  • 与 PyTorch DataLoader 配合使用，自动将单个样本字典合并为批量数据，方便训练和推理。
  • LeRobotDataset 支持流式数据集模式，通过简单修改实现大规模数据集的高效流式处理，适用于需要高性能批处理和帧随机化的 BC 算法。
